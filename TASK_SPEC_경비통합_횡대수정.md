# AXIS 플랫폼 수정 명세서

## 작업 1: 경비 통합 (장비비 + 운반비 → 경비)

### 현재 문제
- 단가비교 탭에서 `장비/기초`, `운반비`가 따로 보임
- 운반비가 1,023,000원으로 높게 나오는데, 내역이 안 보여서 왜 그런지 파악이 안 됨
- 장비비 계산도 정확한지 확인 필요

### 수정 내용

#### 1-1. 단가비교 탭 UI 변경
**파일**: `apps/web/src/pages/quote/Compare.tsx` (또는 단가비교 탭이 있는 컴포넌트)

현재:
```
장비/기초    715,000원
운반비     1,023,000원
```

변경 후:
```
경비      1,738,000원  [▼ 상세보기]
```

`[▼ 상세보기]` 클릭 시 아코디언/드롭다운으로 펼쳐짐:
```
경비      1,738,000원  [▲ 접기]
  ├ 굴착기(0.4m3) 1대     715,000원
  ├ 5t카고 3대 x 170,500원  511,500원  (편도)
  └ 5t카고 3대 x 170,500원  511,500원  (복로-바이백)
```

#### 1-2. 경비 상세 표시 항목
- **장비**: 굴착기 대수, 단가
- **운반**: 트럭 대수, 대당 단가, 편도/복로 구분, 거리 기반 계산근거
- 바이백이면 왕복(x2), 구매면 편도(x1)인 것도 표시

#### 1-3. 경비 계산 검증
운반비 `calcTransport(dist, len, isBB)` 함수 확인:
- trucks = max(2, ceil(len/90))
- perTruck = 130,000 + dist × 900
- trips = 바이백이면 2, 구매면 1
- 현재 공식이 맞는지 엑셀 엔진 VBA(`modTransport_v3_patch.bas`)와 대조 필요

장비비 확인:
- 현재 굴착기 715,000원 고정인데, 엑셀에서는 장비임대료표에서 VLOOKUP으로 가져옴
- 고정값이 맞는지, 높이/연장에 따라 변하는지 확인

## 작업 2: 횡대(표준형) 표시 오류 수정

### 현재 문제
높이 3M 선택 시 XBAR 영역에 `횡대(표준형): 4단`으로 표시됨.
**확정된 규칙에 따르면 3M일 때 표준형도 3단**이어야 함.

### 확정 규칙 (절대 변경 불가)

| 높이 | 실전형(기본) | 스틸예외 | 표준형(구조보강) |
|------|------------|---------|---------------|
| 1M | 2단 | — | 2단 |
| 2M | 2단 | — | 2단 |
| 3M | 3단 | — | **3단** ← 현재 4단으로 잘못 표시됨 |
| 4M | 3단 | 4단(스틸) | 4단 |
| 5M | 4단 | — | 5단 |
| 6M | 5단 | — | **5단** ← 엔진이 6으로 잘못 계산 |
| 7M | 6단 | — | 7단 |
| 8M | 7단 | — | 8단 |

### 원인 분석
엔진 `packages/engine/src/index.ts`의 `getFinalHwangdae()`:
```typescript
const structBonus = (isStd && h >= 4) ? 1 : 0;
```
이 공식은 h>=4부터 +1을 더하는데:
- 3M: structBonus=0 → 3단 (엔진은 맞음)
- 6M: base=5, structBonus=1 → 6단 (엔진이 틀림! 정답은 5단)

**문제가 2곳일 수 있음:**
1. **엔진 함수** — 6M에서 틀림 (단순 공식이 아니라 룩업 테이블 필요)
2. **프론트엔드 XBAR** — 3M에서 4단으로 보이는 건 프론트가 별도로 계산하거나 하드코딩하고 있을 수 있음

### 수정 방법

#### 2-1. 엔진 수정 (`packages/engine/src/index.ts`)
`getFinalHwangdae()` 함수를 룩업 테이블 방식으로 변경:

```typescript
// 표준형 횡대 확정 테이블
const STD_HWANGDAE: Record<number, number> = {
  1: 2, 2: 2, 3: 3, 4: 4, 5: 5, 6: 5, 7: 7, 8: 8,
  // 9M, 10M은 확인 후 추가 (현재 미확정)
};

export function getFinalHwangdae(h: number, panel: string, isStd: boolean, dustN: number): number {
  if (isStd) {
    const stdBase = STD_HWANGDAE[Math.floor(h)] ?? (Math.floor(h) + 1);
    // 스틸 예외는 실전형에만 적용 (4M 스틸)
    return stdBase + dustN;
  }
  const base = getBaseHwangdae(h, panel);
  return base + dustN;
}
```

#### 2-2. 프론트엔드 XBAR 수정
**파일**: 높이 선택 페이지 (XBAR 표시 부분)
- `횡대(표준형)` 값이 어디서 오는지 추적
- 엔진의 `getFinalHwangdae(h, panel, true, dustN)`을 호출하고 있는지 확인
- 별도 하드코딩이나 다른 공식을 쓰고 있다면 엔진 호출로 통일

#### 2-3. 높이 버튼 아래 단수 표시
각 높이 버튼(1M~10M) 아래에 보이는 `2단`, `3단` 등도 확인:
- 이 값들이 실전형 기준인지 표준형 기준인지 명확히
- 엔진 함수와 일치하는지 대조

## 주의사항
- 9M, 10M 표준형 횡대는 아직 미확정 → 수정하지 말 것
- 횡대 규칙은 메모리 파일 `project_hwangdae_rule.md`에 저장되어 있음
- 엔진 수정 후 반드시 테스트 실행하여 1M~8M 전 구간 검증할 것
